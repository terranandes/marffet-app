# uv Migration Finalization Plan

## Goal
Ensure all Python management flows use `uv` exclusively. Update scripts and documentation to reflect this change, eliminating legacy `pip` commands from development workflows.

## User Review Required
> [!IMPORTANT]
> This change deprecates the "Standard Pip" setup for local development. `uv` will be the required tool for running the project locally. Deployment to Zeabur/Render will still use `requirements.txt` generated by `uv pip compile`.

## Proposed Changes

### Scripts
#### [MODIFY] [start_app.sh](file:///home/terwu01/github/martian/start_app.sh)
- Remove `source .venv/bin/activate`.
- Remove `pip install ...` checks.
- Replace `python3 -m venv` with `uv sync`.
- Replace `uvicorn ...` with `uv run uvicorn ...`.

#### [MODIFY] [run_martian.sh](file:///home/terwu01/github/martian/run_martian.sh)
- Remove `source .venv/bin/activate`.

### Documentation
