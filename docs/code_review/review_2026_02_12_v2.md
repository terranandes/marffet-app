## Critical Findings

### 1. `market_data_service.py` — Dividend Logic (Lines 751-766)
```python
if split_val > 0 and split_val != 1.0:
    div_results[y][code]['stock'] += (split_val - 1.0)
```
**Assessment**: Correct for Taiwan market where stock dividends appear as splits (1.1 = 10%). True 2:1 splits (2.0 → 1.0) also convert correctly. The `!= 1.0` filter handles no-op splits.

**⚠️ Edge Case (Low Risk)**: If yfinance reports a *reverse split* (e.g., 0.5), this would add a negative stock dividend (-0.5). Reverse splits are extremely rare in Taiwan market. Acceptable for now.
