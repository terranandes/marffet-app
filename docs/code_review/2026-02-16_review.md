## 1. BUG-111 Analysis

### Observation
The Jira ticket `BUG-111` claims 500 errors on `/api/portfolio/*` when hitting the Next.js frontend (port 3001).

### Findings
1.  **No Conflicting Routes**: I verified that `frontend/src/app/api/` does **NOT** exist. This rules out the hypothesis that a local Next.js API Route is intercepting the request and crashing.
2.  **Rewrite Logic**: `next.config.ts` properly defines the rewrite:
    ```typescript
    source: "/api/:path*",
    destination: `${API_URL}/api/:path*`, // API_URL default: http://127.0.0.1:8000
    ```
3.  **Hypothesis**:
    - The 500 error likely comes from the **Backend** (`martian-api`) itself, which properly receives the proxied request but crashes.
    - **OR**: The Backend is not running/reachable at `127.0.0.1:8000`, and Next.js throws 500 when the proxy connection fails (ECONNREFUSED).

### Recommendation for [UI]/[CODE]
- **Check Backend Logs**: Look at the `martian-api` console. Does it show the request?
    - If YES: The backend crashed. Debug Python.
    - If NO: Next.js failed to connect. Check `API_URL` env var and ensure backend is on port 8000.
- **Test Direct**: Curl `http://localhost:8000/api/portfolio/groups`. If concise, the issue is strictly the proxy.
