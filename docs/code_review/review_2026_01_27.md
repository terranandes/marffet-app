# Code Review: Auth & DB Recovery
**Date:** 2026-01-27
**Reviewer:** [CV] Code Verification Manager

## 1. Database Schema (`app/portfolio_db.py`)
- **Change:** Added `ALTER TABLE` statements inside `init_db`.
- **Review:**
    - **Safety:** Uses `try/except` to ignore "Duplicate Column" errors. Safe.
    - **Performance:** `ALTER TABLE` is O(1) in SQLite. Safe for startup.
    - **Verdict:** **APPROVED**. Clever "Self-Healing" pattern for persistent volumes without heavyweight migration tools (Alembic).

## 2. Authentication Logic (`app/auth.py`)
- **Change 1:** `nuke_cookies` now imports `COOKIE_DOMAIN` from `main.py`.
    - **Review:** Ensures deletion logic matches creation logic. crucial for `localhost` vs `None`. **APPROVED**.
- **Change 2:** `login` uses `Referer` to determine `redirect_uri`.
    - **Review:** Necessary for Dual-Domain Zeabur setup. Logic seems robust (`frontend_base in referer_base`).
    - **Security:** Limits redirection to known `FRONTEND_URL` or internal generation. **APPROVED**.
- **Change 3:** HTTPS Enforcement (`redirect_uri.replace("http", "https")`).
    - **Review:** Essential for Proxy environments (Zeabur/Nginx). Without this, Google rejects the flow. **APPROVED**.

## 3. Frontend Fetch Logic (`frontend/**`)
- **Change:** `API_URL` / `API_BASE` set to `""` (Empty String).
- **Review:**
    - Usage: `fetch("/auth/me")`.
    - Browser Behavior: Uses current origin (`martian-app`).
    - Result: Cookie (Host-Only) is sent. Next.js proxies.
    - **Verdict:** **APPROVED**. This is the standard pattern for Next.js Proxies. Attempting Absolute URLs was the original design flaw.

## 4. Middleware Order (`app/main.py`)
- **Change:** Moved `ProxyHeadersMiddleware` to the end (Run First).
- **Review:** Critical fix. Without this, `SessionMiddleware` (Secure Cookies) runs on `http` context, causing cookie rejection or "SameSite" issues. **APPROVED**.

## 5. Artifacts
- **Cleanup:** Testing scripts `reproduce_logout.py` removed.
- **Docs:** `product/auth_db_architecture.md` created. Excellent documentation practice.

**Overall Status:** PASS
The codebase is cleaner and more robust than at the start of the session.
