### 4. `supplement_splits.py` — UPSERT Split Patcher

**Verdict**: ⚠️ **APPROVED with critical note**

- Good CLI design with `--apply` flag (dry-run by default)
- UPSERT logic (`ON CONFLICT DO UPDATE SET stock = EXCLUDED.stock WHERE dividends.stock = 0`) is correct
- Sanity cap at `stock_div > 1000` prevents insane values

**Critical Issue**: The `WHERE dividends.stock = 0` clause in the UPSERT means that if a stock already has a non-zero stock dividend for that year, the update is silently skipped. This caused **0050's 2025 split to not be patched** (existing record had `stock=0.0` but the update didn't fire — likely due to the first INSERT attempt creating a row with `cash=0.0` that conflicted). Required a manual fix.

**Root Cause**: The UPSERT inserts `cash=0.0` on conflict, which could overwrite the existing cash dividend if the WHERE clause matched. The logic is subtly wrong for mixed events. A safer approach:
```sql
UPDATE dividends SET stock = ? WHERE stock_id = ? AND year = ? AND stock = 0
```
Followed by INSERT only if UPDATE affected 0 rows.
